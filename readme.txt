Нужно поменять в inventory айпишники на ваши удалённые хосты. На удалённых хостах создать пользователя ansible с паролем ansible и включить ssh сервер на порту 22
Если вы пересобираете ansible-playbook ещё раз, то в init.sql нужно добавить строчку 
select pg_drop_replication_slot('replication_slot');
Так как при первом запуске создаётся replication_slot, то при втором запуске, если не удалить нынешний replication_slot возникнет конфликт

Нужно создать файл .env, который будет отвечать за то, чтобы ansible успешно отрабатывал.
[ПРИМЕР .env файла]
RM_HOST: '192.168.1.105'
RM_PORT: '22'
RM_USER: 'root'
RM_PASSWORD: 'Qq123456'

DB_USER: 'postgres'
DB_PASSWORD: 'Qq12345'
DB_HOST: '192.168.1.101'
DB_PORT: '5432'
DB_DATABASE: 'pt_db'

DB_REPL_USER: 'postgres'
DB_REPL_PASSWORD: 'Qq12345'
DB_REPL_HOST: '192.168.1.103'
DB_REPL_PORT: '22'
==============================================================================================================================================
